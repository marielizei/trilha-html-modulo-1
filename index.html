Evolução do computador

Os computadores têm uma linha do tempo dividida por gerações. Cada uma delas é marcada por características definidas por inovações ou meios de funcionamento em específico. Entretanto, a principal característica que define uma geração de computadores é a capacidade e velocidade de processamento. Veja, a seguir, as gerações e quais foram os seus modelos mais importantes:

Primeira geração (1939-1955)

Os primeiros computadores modernos diferiam-se de suas pré-gerações por utilizarem tubos a vácuo que substituíam os relés. Por meio desses tubos, as informações binárias processadas pelos computadores ocorriam de maneira mais dinâmica. Mediam metros de altura e comprimento, pesavam toneladas e tinham um custo que seria atualmente milionário. O Eniac, por exemplo, criado na década de 1940, media cerca de 180 m2 e pesava quase 30 toneladas.

Segunda geração (1955-1964)

Agora, o processamento de dados ocorria por meio de transistores , que controlavam o fluxo da corrente elétrica por meio do chaveamento das informações binárias. Custavam menos que tubos a vácuo, geravam menos calor, eram ligeiramente menores e tinham maior durabilidade; tinham também a capacidade de compreenderem linguagem de programação mais complexa, como a Fortran e a Cobol

Terceira geração (1964-1974)

Introduziu nos computadores o uso de chips, ou circuitos integrados, que poderiam processar informações simultâneas de diversos transistores. Como consequência, esses computadores eram muito mais rápidos e baratos, e foram distribuídos em empresas de menor proporção.

Quarta geração (1974-1989)

Nessa fase, o computador funcionava por meio de microprocessadores que suportavam milhares de transistores. Tal período permitiu o desenvolvimento de modelos compactos e introduziu os microcomputadores de uso doméstico. Devido às múltiplas opções de programação, houve a criação dos primeiros jogos eletrônicos comercializáveis e o avanço da internet.

Quinta geração (1989-presente)

A atual geração de computadores foi marcada pela introdução do ULSI, tecnologia que permitia a inserção de milhões de transistores por chip. As múltiplas funções simultâneas permitiram o uso de modelos mais rápidos e de aquecimento consideravelmente menor. Essa fase significou o boom do consumo de microcomputadores, aumentando as vendas de seu mercado para 60 milhões de unidades em 1995.

O que são Clients?

(voltar)

É o cliente quem sempre inicia a comunicação com o servidor. O cliente mais conhecido é o navegador. Quando acessamos um site pelo navegador, uma requisição é disparada pela internet para o computador no qual esse site está hospedado.

Chamamos as aplicações que agem em nome do usuário, assim como o navegador, de user-agent. Após enviar uma requisição, o user-agent aguarda até que haja uma resposta do servidor e então a comunicação se encerra.

O que são Servers?

(voltar)

O papel do servidor web é receber uma requisição e devolver uma resposta para o cliente. Geralmente o servidor não envia dados se o cliente não disparar primeiro uma requisição. O servidor é um ou mais computadores com um único endereço. Nestes computadores podem ser hospedados documentos, imagens, áudio, vídeo, etc.

Geralmente o servidor também hospeda programas que atendem as requisições dos clientes e as processam. Chamamos esses programas de web server, ou servidor web. É com um web server que nos comunicamos, sendo o computador no qual ele está instalado parte da infraestrutura que o conecta a internet. O web server geralmente é capaz de executar aplicações escritas em linguagens como PHP, C#, Java, etc.

Como é uma estrutura básica de um HTML?

(voltar)

Antes de falarmos sobre a estrutura básica do HTML, vamos entender o que é o HTML e como ele funciona

O HTML é uma linguagem de marcação de HiperTexto utilizada para estruturar um documento ou página para a web. Ou seja, ao visualizar uma página na internet, você verá a renderização de um código criado em HTML em conjunto com outras linguagens.

Um documento HTML nada mais é que um documento contendo a extensão .html ou .htm. A partir deste documento, o navegador faz a leitura do arquivo e renderiza o seu conteúdo para que o usuário final possa visualizá-lo. Portanto, os arquivos .html ou .htm podem ser visualizados em qualquer navegador (como Google Chrome, Opera, Safari, Mozilla Firefox, etc).

Geralmente um site é composto por diversas páginas HTML, como por exemplo: um website que contenha três páginas (uma homepage, uma página de contato e uma página de produtos) receberá ao menos três documentos .html distintos, sendo um para cada página do website.

Estrutura básica de um documento HTML

A estrutura básica de um documento HTML é realizada da seguinte forma:

< !DOCTYPE html>
< html>

< head>
< meta charset="UTF-8">
< title> titulo da pagina< /title>
< /head>

< body>

< /body>

< /html>

A partir desse código, podemos desenvolver toda a nossa página, estruturando como precisarmos. Mas primeiro, é bom compreender o que está sendo feito até o momento:

  • DOCTYPE html — A declaração do doctype informa ao navegador que está sendo utilizado a linguagem HTML . Hoje em dia, o doctype basicamente é necessário apenas para garantir que o documento se comporte corretamente.
  • html — O elemento html envolve todo o conteúdo da página HTML.
  • head - O elemento head é o local no qual você irá incluir todos os elementos que deseja incluir na página mas que não deve ser renderizado. Dentre eles, podemos utilizar os metadados, o título, as tags link para adicionar folhas de estilos, entre outras.
  • meta charset=”utf-8″ — Esse elemento de metadados serve para definir o conjunto de caracteres que seu documento deve usar para o utf-8.
  • title — O elemento title serve para definir o título de sua página. Portanto, é aqui que se define o título que é mostrado na guia do navegador.
  • body — Finalmente, neste elemento, ficarão todos os elementos que serão renderizados pelo navegador, ou seja, todo o corpo da sua página HTML.

Entendendo a diferença entre listas ordenadas e não ordenadas

(voltar)

As listas não ordenadas, também conhecidas como listas com marcadores, são usadas quando a ordem dos itens não é importante. Elas são comumente usadas para listar itens de forma geral, como uma lista de ingredientes em uma receita, uma lista de recursos em um produto ou uma lista de tarefas.

Já as listas ordenadas, também conhecidas como listas numeradas, são usadas quando a ordem dos itens é importante. Elas são úteis em uma variedade de situações, como criar uma lista de instruções passo a passo, uma lista de classificação ou qualquer lista onde a ordem dos itens é significativa.

Veja a seguir um exemplo utilizando os dois tipos de lista:

Receita de Focaccia

ingredientes:

  • 10g fermento biológico
  • 1 colher de sopa açúcar
  • 1 colher de sobremesa sal
  • 2 colheres margarina
  • 1 copo (200 ml) leite
  • 1 copo (100 ml) H2O água
  • 600g farinha de trigo
  • 1 calabresa
  • 1 cebola
  • 200g muçarela em cubo
  • 200g de presunto em cubo
  • 50g azeitona
  • Azeite a gosto

modo de preparo:

  1. Na bacia coloca os ingredientes seco, e aos poucos vai adicionando a água e o leite (morno)
  2. Mistura bem até desgrudar das mãos e ficar linsinha.
  3. Deixe descansar por 40 min. Ou até dobrar de tamanho.
  4. Abra a massa, coloque no tabuleiro, recheie, deixa descansar mais 30 minutos.
  5. Regue azeite e asse em forno médio por 30 minutos.
Observação: A Focaccia é um pão de origem italiana (Gênova), achatado e bem macio e suave - Receita disponível no site da Cookpad
